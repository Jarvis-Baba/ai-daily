## 一、项目意图（推断）

AI 行业每日简报自动生成器，运行于 WSL Ubuntu，每天 07:00 由用户级 systemd timer 触发。链路：

多源抓取（RSS/博客/GitHub/SEC/Reddit/signal-inbox）→ L0 原始工件捕获 → L1 证据编译（LLM 提取结构化事实）→ 事件聚类 → 两轮 LLM 评分（Impact/Novelty/Actionability 加权加法）→ Alpha/Beta/Gamma 分桶 → 跨日 Theme Memory → LLM 合成 6 板块简报 → Markdown + Content IR JSON → 外部 Visual Compiler 生成配图 → 复制到 E:\Jarvis\Outputs 交付。

LLM 用 DeepSeek（OpenAI SDK 兼容）。架构是编译器隐喻（L0→L1→L2→L3 分层、ABI 冻结、Artifact/Evidence/Package 三层分离）。文档体系（PROJECT/STATE/CHANGELOG/DECISIONS）是多 agent 交接用的 Project OS。另有独立 TypeScript dashboard（运营观测台），未接入每日环路。

## 三、问题清单（按严重度）

### 🔴 高

1. **[静态]** 生产代码大量未入版本控制（见上）。与项目第一原则 "Files are source of truth" 直接矛盾，目录损坏即无法恢复。

### 🟡 中

3. **[静态+日志佐证]** 遥测日期错位一天：structural_telemetry.py:184 用 UTC 日期命名指纹文件，main.py 用本地日期（CST）作 report_date。07:00 CST = 前一日 23:00 UTC，故 06-10 报告对应 structural_fingerprint_20260609.json。Dashboard 按日期 join 会错位。
4. **[静态]** run.sh 错误处理失效：开头 set -e 使 `EXIT_CODE=$?` / `PIPELINE_EXIT=$?` 永远只能是 0；任何失败直接退出，后面的 Output Contract Enforcement（FATAL 检查、E 盘交付）在失败路径根本不执行。
